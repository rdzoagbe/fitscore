# P6C SEO QA Checklist

Run this checklist after every public SEO/content deployment.

## Crawlability

- `/resources` loads.
- `/roles` loads.
- Every resource guide loads.
- Every role page loads.
- `/sitemap.xml` includes all public SEO URLs.
- `/robots.txt` blocks `/admin` and `/api`.

## Metadata

- Each public page has one H1.
- Each public page has a unique title.
- Each public page has a unique meta description.
- Canonical URLs point to `https://joblytics-ai.com/...`.
- JSON-LD is valid and does not include private user data.

## Copy quality

- Page copy explains a real job-search decision.
- No page claims guaranteed interviews or guaranteed ATS ranking.
- No page suggests LinkedIn password collection or private LinkedIn scraping.
- Role pages mention the specific role in the H1, CTA, and at least two content sections.
- CTAs use action language: “Start a free ATS check,” not vague language like “Get started now.”

## Mobile UX

- Hero CTA is visible without excessive scrolling.
- Cards stack cleanly.
- Long copy does not overflow.
- Tap targets are comfortable.
