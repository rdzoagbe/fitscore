# P7D — Mobile UX Polish

This sprint adds a responsive CSS layer that improves readability, form comfort, dashboard/card spacing, mobile CTA behavior, and horizontal usability for dense boards such as the application pipeline.

## Pages to check manually

- `/`
- `/pricing`
- `/dashboard`
- `/linkedin`
- `/resources`
- `/roles`
- `/sample-reports`
- `/early-access`
- `/admin`
- `/admin/reliability`

## Mobile QA checklist

1. No horizontal page overflow on 360px width.
2. Analyzer form is easy to fill from a phone.
3. Textareas do not feel cramped.
4. Dashboard cards stack cleanly.
5. Pipeline columns scroll horizontally and are usable.
6. Pricing cards and tables do not break layout.
7. CTA buttons are large enough for touch.
8. Modal/onboarding content is scrollable.
9. Admin pages remain readable on mobile.
10. Strict QA and build still pass.
